 Sutra 📦 Update Introduces Insights Hub API and VS Features #1 ▾
This release ships a new API contract for the Karsa Insights Hub and introduces significant client-facing features for Verisimilitude Score (VS), Discovery, and Quick Peek functionality. ### Summary of Changes * **Introduces Insights Hub API Contract:** Documents the new unified API for Authority Score (AS) industry benchmarks, combining measured and estimated domain data. This contract lays the foundation for future analytics and VS endpoints. * **Implements Verisimilitude Score (VS) Features:** Adds comprehensive client routes for configuring and accessing VS data, including AI platform management, historical snapshot viewing, detailed probe analytics, and platform override settings for fine-grained control. * **Adds Discovery and Focus Management:** Introduces new client endpoints to enable content discovery, manage strategic content 'focuses', trigger discovery runs, apply recommended focuses, and generate/manage prompt briefs and individual prompts. * **Integrates Quick Peek Functionality:** Provides a new client endpoint (`/api/client/domains/quick-peek`) to retrieve Authority Signal scores for any domain, complete with a 7-day cache and monthly usage quotas enforced per tier. * **Enhances User Account Management:** Enables clients to update their primary domain and user profile pictures, including validation for image inputs. * **Hardens Authentication:** Corrects email verification and magic link URLs to prevent hostname misconfiguration behind proxies and refines cookie handling for better compatibility with modern deployment environments. * **Updates Tier Configuration:** Adds new feature flags (`pipeline_detail`) and usage limits (`quick_peek_monthly`) to the tiering system, allowing granular control over access to the new functionalities. * **Enables Client Domain Removal:** Allows clients to remove domains from their organization through a new endpoint, ensuring proper access revocation and cache invalidation.
